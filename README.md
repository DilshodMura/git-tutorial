# Rules

- Always make sure you ONLY check in things YOU changed
- Make sure your solution to the problem is isolated in different method/function
- Make sure to write at least 3 happy path unit tests and 3 unhappy path unit tests
- Make sure to remove commented code, leaving comments are fine, but commenting code is bad
- Optimize your code as much as you can, i.e. extract repeating code in its own method/function
- Follow all naming rules for variables, functions/methods, classes/structs/interfaces etc.
- Ask questions
